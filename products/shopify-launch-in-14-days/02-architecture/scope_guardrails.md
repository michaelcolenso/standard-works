# Scope Guardrails

## Must include
- A strict 14-day plan with daily deliverables.
- Explicit dependencies (what must happen before the next step).
- At least 5 reusable assets that reduce execution friction.
- Pre-launch and first-week post-launch operating procedures.

## Must avoid
- Deep advanced modules (international tax complexity, multi-channel attribution, full CRO systems).
- Unbounded app roundups without clear use-case criteria.
- Generic motivational content not tied to execution actions.

## Decision filters
Include content only if it satisfies at least one:
1. Prevents a likely launch failure.
2. Increases conversion trust/readiness before first sale.
3. Improves diagnostic ability in first 7 days post-launch.

## Change control
Any new section request must state:
- the failure mode it addresses,
- expected buyer outcome,
- what existing content it replaces to preserve scope.
