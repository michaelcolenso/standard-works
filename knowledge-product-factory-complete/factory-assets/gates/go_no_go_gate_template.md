# Go / No-Go Gate (Template)

Purpose:
Prevent the buyer’s most expensive early mistake with a binary gate.

Instructions:
Answer each question Yes/No.
- 0–1 No → GO
- 2–3 No → WAIT
- 4+ No → ALTERNATE PATH

Checklist (customize per product):
1) Goal defined in one measurable sentence
2) Inputs complete and verified
3) Top risks identified with mitigations
4) Path fits constraints (time/budget/authority)
5) Dependencies satisfied
6) Rollback plan exists
7) Downside bounded (max acceptable loss)
8) Verification evidence defined
9) Common trap ruled out
10) Next step executable within 48 hours

Output:
Decision + top gaps.

Verification:
List the evidence proving readiness.
